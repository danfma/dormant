# Dormant Documentation

This documentation is for developers evaluating, trying, or contributing to Dormant. It is intentionally plain Markdown so it works in the repository and on GitHub without a documentation-site build step.

## Reading Paths

New to the project:

- Start with the [README](../README.md).
- Check the [capability status](status.md).
- Follow [getting started](getting-started.md).

Trying Dormant in code:

- Read [getting started](getting-started.md).
- Learn the [DormantQL schema language](guides/dormantql-schema.md).
- Learn [queries and mutations](guides/queries-and-mutations.md).
- Check [naming and generated code](guides/naming-and-generated-code.md).

Reviewing or contributing:

- Read [architecture](architecture.md).
- Read [design decisions](design-decisions.md).
- Use [SpecKit sources](speckit-sources.md) to trace claims back to the design artifacts.
