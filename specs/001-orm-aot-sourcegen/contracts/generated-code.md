# Contract: Generated-code

The shape of emitted code is a compatibility surface (Constitution II): consumer code binds to it, so it
MUST NOT change incompatibly within a MAJOR version. The committed Verify `.verified.txt` snapshots are the
baseline; any change is a reviewable diff (research §8).

## What the generator emits

Per schema entity `E` (in namespace `PascalCaseEachPart(rootNamespace + folders + module)`, FR-046):
- `public partial class E` — mapped properties (auto-props) + link members typed `Link<T>`/`LinkSet<T>`.
  Non-nullable members use the C# `required` modifier; nullable members use `T?` and omit `required`
  (FR-047/FR-048). Hand-written `partial` members in separate files coexist and survive regeneration
  (FR-003).
- Internal `static` `[UnsafeAccessor]` field accessors for `E`'s mapped members (materialization +
  snapshot), no reflection (research §5).
- A `readonly record struct E_Snapshot` + a `E_Diff` comparer (changed-column detection, FR-014).
- A `RowMaterializer<E>` using `GetFieldValue<T>` per column, ordinals fixed at build time (no boxing).

Per projection shape `P`:
- `public sealed partial class P` (or record) with exactly the requested fields + nested projection types
  (FR-007/FR-008). No setters for non-requested fields — they do not exist.

Per query `Q`:
- A typed method (e.g. `ISession.Q(...)` partial/extension) returning the result type, carrying a
  `CompiledQuery<TResult>` with the **prebuilt SQL** (and optional-param fragments) (FR-013/FR-031).

Per jsonb-mapped type:
- A `[JsonSerializable]` `partial JsonSerializerContext` and the (de)serialize calls (research §3).

Per native binding/function:
- An `ITypeBinding<T>` registration and/or a typed invocation wrapper, provider-scoped (FR-038/FR-039).

## Stability rules

- **Additive only** within a MAJOR: new generated members/types are fine; renames/removals/visibility
  reductions of previously-emitted members are breaking and require a MAJOR + migration note.
- **Deterministic**: identical DSL ⇒ byte-identical output (ordinal ordering, invariant formatting,
  stable hint names) — enforced by cacheability + Verify tests (FR-004).
- **No runtime reflection / no runtime query compilation** in any emitted code (FR-013/FR-017).
- Generated files are marked `// <auto-generated/>`, `#nullable enable`, and namespaced to avoid collision
  with hand-written partials.
