# Contract: Generated code (immutable entities + command/query methods)

The emitted code is a compatibility surface (Constitution II): consumer code binds to it, so it MUST NOT
change incompatibly within a MAJOR. Verify `.verified.txt` snapshots are the baseline; any change is a
reviewable diff.

## Per schema entity `E` (namespace `PascalCaseEachPart(rootNamespace + folders + module)`)

- `public sealed partial record E` (or class with init-only members) — mapped value members + read-side
  relationship members (`Ref<T>` / `RefSet<T>` / …). **Immutable**: init-only/positional, **no public
  setters**, no tracked/snapshot state. Relationships default to the `Unloaded` sentinel.
- A no-reflection materialization constructor (reads columns by ordinal via the generated ctor, no
  `[UnsafeAccessor]`), producing the immutable instance.
- Identity equality by primary key (carried from `001`); records also get structural value semantics.

## Per authored command `C` (in `partial static {Module}Commands`, C# 14 extension block on `ISession`)

- A typed method (e.g. `session.CreateUser(email, ct)`) returning the command's **immutable** result
  (inserted entity / projection / affected-count signal).
- The method uses a **reused compiled definition** (a single static readonly per method): prebuilt
  **CTE SQL** (for nested/related writes) + a no-boxing parameter binder + a materializer for any `returning`
  shape. The definition is allocated **once** and reused across calls (SC-007); only parameter values vary.
- Nested/related writes render as one `WITH … RETURNING … <final>` statement → one round-trip.
- `update`/`delete` with a concurrency-token match surface a conflict when zero rows are affected.

## Per authored query `Q` (carried from `001`)

- A typed `ISession` method in `partial static {Module}Queries` (C# 14 extension block) returning an
  immutable full entity or a distinct projection record; optional parameters via fragment selection; build-
  time SQL on a reused compiled definition.

## DDL / jsonb / naming (carried from `001`)

- Schema-qualified `CREATE SCHEMA`/`CREATE TABLE` DDL applied via the schema initializer; configurable
  naming convention (snake_case default) + per-unit `db("…")` overrides; `jsonb` value columns emit a
  `::jsonb` write cast.

## Stability rules

- **Additive only** within a MAJOR; renames/removals of emitted members are breaking.
- **Deterministic**: identical DQL ⇒ byte-identical output (ordinal ordering, invariant formatting), enforced
  by Verify + cacheability tests.
- **No runtime reflection / no runtime query or command compilation** in emitted code; all SQL is build-time.
- Generated files: `// <auto-generated/>`, `#nullable enable`, namespaced to coexist with hand-written
  partials.

## Removed vs `001`

No mutable entity setters; no snapshot type; no change-tracking/diff code; no `AddAsync`/`Remove` emission.
Writes exist only as generated command methods.
